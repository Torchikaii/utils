# Create Project Requirements Document (PRD)

## Purpose
Generate a comprehensive Project Requirements Document that outlines project scope, objectives, technical requirements, and success criteria.

## Instructions

### 1. Project Overview
- **Project Name**: Clearly state the project name
- **Description**: Brief summary of what the project does
- **Objectives**: List primary goals and outcomes
- **Scope**: Define what is included and excluded from the project

### 2. Requirements
- **Functional Requirements**: What the system must do
  - User stories or use cases
  - Feature descriptions
  - System behaviors
- **Non-Functional Requirements**: Quality attributes
  - Performance requirements
  - Security requirements
  - Scalability requirements
  - Reliability/availability targets

### 3. Technical Specifications
- **Technology Stack**: Languages, frameworks, tools
- **Architecture**: High-level system design
- **Dependencies**: External libraries, services, APIs
- **Constraints**: Hardware, software, or environmental limitations

### 4. Success Criteria
- **Metrics**: How success will be measured
- **Acceptance Criteria**: Conditions that must be met
- **Performance Targets**: Specific benchmarks

### 5. Timeline & Resources
- **Milestones**: Key project phases and dates
- **Team**: Required roles and responsibilities
- **Budget**: Resource allocation (if applicable)

### 6. Risks & Assumptions
- **Assumptions**: What we assume to be true
- **Risks**: Potential obstacles and mitigation strategies
- **Dependencies**: External factors that could impact the project

### 7. Deliverables
- List all tangible outputs
- Documentation requirements
- Testing and validation deliverables

## Format
Use markdown format with clear sections and subsections. Include tables where appropriate for requirements matrices or timelines.

## Output
Save as `PRD.md` in the `.opencode` directory.
